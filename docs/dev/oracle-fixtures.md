# f1db oracle fixture format (v2)

One JSON file per season at `tests/Companion.Tests/Fixtures/f1db/<year>.json`, generated by
`tools/Companion.FixtureGen` from the f1db SQLite release plus `data/rules/f1-points-systems.json`.
Fixtures are self-contained: catalog round overrides (half points, Indy constructors exclusion,
alternate tables) are resolved to per-round flags at generation time.

```jsonc
{
  "year": 1967,
  "roundCount": 11,                     // races that season per f1db
  "rounds": [
    {
      "round": 1,
      "grandPrixId": "south-africa",   // f1db grand prix id, for traceability
      "pointsFactor": "1",             // "1/2" half points, "2" double points (race sessions only)
      "countsForConstructors": true,   // false: Indy 500 1958-60
      "alternateRaceTableId": null,    // 2022+ shortened-race table key, when used
      "sessions": [
        {
          "kind": "race",              // "race" | "sprint"
          "fastestLapDriverIds": ["denny-hulme"],   // ALL holders of the fastest time (ties!)
          "entries": [
            {
              "driverId": "pedro-rodriguez",
              "constructorId": "cooper+maserati",  // see Constructor identity below
              "position": 1,           // null when unclassified
              "status": "classified",  // classified|retired|disqualified|notClassified|didNotStart|excluded
              "sharedDrive": false,
              "pointsEligible": true,  // false: classified but officially no points (see below)
              "pointsPosition": null   // set when official points used a different rank (see below)
            }
          ]
        }
      ]
    }
  ],
  "expectedDrivers": [
    { "id": "denny-hulme", "position": 1, "points": 51.0 }   // position null when f1db has none (EX/excluded)
  ],
  "expectedConstructors": [                                   // omitted for seasons before 1958
    { "id": "brabham+repco", "position": 1, "points": 63.0 }
  ]
}
```

## Mapping rules (FixtureGen)

- **Entries**: include entrants who took part in the race. Map f1db `positionText`:
  numeric → `classified` with that position · DNF/retirements → `retired` · `DSQ` →
  `disqualified` · `NC` → `notClassified` · `DNS` → `didNotStart` · `EX` → `excluded`.
  `DNQ`/`DNPQ`/`DNP` (never started the race) are dropped entirely.
- **Constructor identity**: the championship entity is chassis+engine ("Lotus-Climax" and
  "Lotus-BRM" scored separately). Key: `constructorId + "+" + engineManufacturerId`, matching
  f1db's `season_constructor_standing` keying.
- **Entity splits** (2018 Force India): the catalog's `constructorEntitySplits` renames the
  entity from `fromRound` onward (rounds ≥ 13 → `racing-point-force-india+mercedes`); f1db's
  two same-key standing rows map to old id (the 0-point excluded row) and `newId` (the scoring
  row) respectively.
- **Points eligibility** (derived from f1db's per-row `race_points`, never authored):
  - `pointsEligible: false` when a row is classified at a position within the season's table
    depth but `race_points` is NULL/0 — F2 cars in the 1958/1967/1969 German GPs, unregistered
    second cars (1984 Monza, 1987 Adelaide), annulled results (Hill, 1963 France). Skip rows in
    shared-drive groups (their NULL points mean the shared-drive policy, not ineligibility).
  - `pointsPosition: k` when a row's `race_points` exactly equals the season table's value for
    a DIFFERENT position k (1967/1969 German GPs: F1 cars paid by rank among eligible cars —
    Bonnier classified 6th, paid as 5th). Apply only on factor-1 rounds, to integer point
    values, on non-shared rows, and never to rows belonging to that race's fastest-lap holders
    (1950s race_points may embed the FL point).
- **Shared drives**: rows f1db flags `sharedCar` → `sharedDrive: true`; all drivers of one car
  carry the same position.
- **Fastest lap**: every driver holding rank 1 in the round's fastest-lap data (1954 British GP
  has seven). Sprint fastest laps are never emitted.
- **Sprints**: f1db sprint results become a second session `"kind": "sprint"`.
- **Expected standings**: straight from f1db `season_driver_standing` /
  `season_constructor_standing` (points as the raw REAL value).

## Comparison rules (oracle test)

- Every expected driver/constructor must appear in engine output with
  `|engine.CountedPoints.ToDouble() − expected.points| < 0.01`
  (tolerance because f1db stores REAL; 1/7-point fractions may be rounded).
- **Positions compare tie-tolerantly.** Official standings conventions vary by era: pre-2000
  dead heats share a position and skip the following ones (1950: four drivers P9, next is
  P13), later eras rank ties countback-style or in scoring-chronology order. Rule: group the
  EXPECTED standings by points value; a competitor's engine position matches when it lies in
  `[groupMinExpectedPosition, groupMinExpectedPosition + groupSize − 1]`. Mismatches across
  different points levels remain hard failures. Null expected positions skip the check.
- Engine competitors absent from the expected list must have zero counted points
  (f1db may omit pointless entrants; the engine never omits anyone who raced).
- Excluded competitors: engine zeroes excluded constructors' counted points (2007 McLaren,
  2018 pre-split Force India) so they compare against f1db's 0-point rows directly; excluded
  drivers (1997 Schumacher) keep points, and only the null position is compared.
- Failure messages must name the season, the competitor, expected vs actual, and the engine's
  dropped-rounds list for that competitor — mismatches are usually rules-data bugs, and the
  dropped list is what you debug with.
